• The root element of an HTML page is represented by the HTML element html, which is also known as the root element. This element must be the ancestor of all other elements.
  • The base URL for all relative URLs in a document is specified by the HTML element known as "base". A document may only contain one base element. Scripts can use Node.baseURI to get the base URL used by a document. BaseURI defaults to location.href in the absence of any base elements in the document.
  • The head HTML element contains machine-readable information (metadata) about the document, like its title, scripts,and style sheets.
  • The HTML element known as link describes connections between the current content and other resources. This element is most frequently used to connect to stylesheets, but it may also be used to create site icons, including "favicon"-style icons and icons for mobile devices' home screens and applications.
  • The HTML element meta is used to express metadata that cannot be represented by other elements related to HTML meta, such as "base," "link," "script," "style," or "title."
  • A document's or a section of a document's style information is contained in the HTML element "style". It includes CSS, which is used to style the text and images in the page that contains the style element.
  • The title of the document that appears in a browser's title bar or on a page's tab is defined by the HTML element title. Tags included within the element are disregarded; it simply includes text.
  • The content of an HTML document is represented by the body HTML element. A document can only have one body element.
  • The HTML element address indicates that the enclosing HTML contains contact information for a person or individuals, or for an organization.
  • A self-contained composition that is meant to be independently distributable or reusable is represented by the HTML element known as article in a document, page, application, or website. An interactive widget or gadget, a forum post, a magazine or newspaper article, a blog entry, a product card, or any other standalone piece of content are examples.
  • A section of a document whose content is only loosely connected to the page's main content is represented by the HTML element aside. Sidebars or call-out boxes are common ways to convey asides.
  • The HTML element footer specifies a footer for its ancestor sectioning content or sectioning root element. A footer often provides information about the section's author, copyright information, or links to related papers.
  • The HTML element header designates introductory information, which is often a collection of introductions or navigational tools. It could also have a logo, a search form, the author's name, and other features in addition to certain header components.
  • Six levels of section headers are represented by the HTML elements h1 through h6. The highest section level is h1, while the lowest is h6.
  • The primary material of a document's body is represented by the HTML element main. The key subject of a document or the primary function of an application is expanded upon or directly connected in the main content section.
  • An area of a page designated as nav in HTML designates a part whose function is to offer navigation links, either inside the current document or to other pages. Menus, tables of contents, and indexes are typical illustrations of navigation sections.
  • A generic freestanding part of a page that lacks a more specialized semantic element is represented by the HTML element section. Only very seldom should a section not have a heading.
